# AGENTS.md

## Purpose
このディレクトリ配下で作業するエージェントの行動基準を定義し、変更品質・安全性・レビュー容易性を担保する。

## Scope
- 本ファイルは、このファイルが配置されたディレクトリ配下すべてに適用する。
- より深い階層にある `AGENTS.md` が優先される。
- 直接のシステム／開発者／ユーザー指示は、本ファイルより優先される。

## Required
- 変更前に、関連するコード・設定・既存ドキュメントを確認する。
- 変更は目的に対して最小限にし、無関係な差分を含めない。
- 変更理由・影響範囲・検証結果を最終報告に明記する。
- テストや確認が実行できない場合は、理由を明記する。
- 既存仕様と挙動差分がある場合は、意図的変更か不具合かを明示する。

## Prohibited
- 秘密情報（APIキー、トークン、個人情報、認証情報）の出力・保存・コミット。
- 履歴改変（`git push --force` など）や破壊的操作の独断実行。
- 根拠のない推測による断定的な説明。
- 大量の自動整形や広範囲リネームなど、レビュー困難化につながる変更の独断実施。

## Coding Rules
- 既存のコードスタイル、命名規則、ディレクトリ構成を優先する。
- 新規依存の追加時は必要性と影響を説明する。
- 変更対象外のリファクタは行わない。
- コメント・ドキュメントは、利用者が実行手順を再現できる粒度で記述する。

## Validation
- 実行可能な範囲で lint / test / 型チェック等を行う。
- 実施したコマンドと結果（成功／失敗／制約）を明示する。
- 失敗時は、再現手順・想定原因・暫定回避策（あれば）を記録する。

## PR Rules
- タイトルは「何を変えたか」が即座に分かる形にする。
- 本文には、目的・主な変更点・検証内容・未解決事項（あれば）を含める。
- 1PR 1目的を原則とし、混在する論点は分割を検討する。

## Communication
- 既定の報告言語は日本語。
- 不確実な点は前提・不確実性を明示し、必要に応じて確認を促す。
- 既知の制約（環境依存・未検証範囲）は、断定を避けて明示する。

## Update Policy
- 本ドキュメントを更新する際は、運用で実際に問題となった点を根拠にする。
- 抽象的なルール追加は避け、実行可能な文言にする。
- 更新時は、変更意図をコミットメッセージまたはPR本文に記載する。
